## 1. 存储概况

当前两台节点各有一块约 54.6 TB 的数据盘，并通过 NFS 互相挂载。整体上，用户可以看到 `/data1` 和 `/data2` 两个主要数据路径。

| 挂载点 | 说明 |
|---|---|
| `/` | 系统盘，约 888 GB，不建议长期存放大规模计算数据 |
| `/data1` | 主要数据目录之一，底层来自其中一台节点的数据盘 |
| `/data2` | 主要数据目录之一，底层来自另一台节点的数据盘 |
| Swap | 每节点约 3.8 GB |

## 2. 使用建议

- 大型输入、输出和中间文件建议放在 `/data1` 或 `/data2`；
- 不建议把大规模计算数据长期堆放在系统盘或个人 home 目录；
- 大文件计算前建议确认磁盘剩余空间；
- 大量小文件会显著增加文件系统压力，建议按任务建立独立目录；
- 重要结果应及时备份到本地或课题组长期存储位置。

查看磁盘空间：

```bash
df -h
```

查看当前目录占用：

```bash
du -sh .
```

查看某个目录下文件大小：

```bash
du -h --max-depth=1 /data1/$USER 2>/dev/null | sort -h
```

## 3. 网络概况

当前节点间通过 1 GbE 以太网互联，未检测到 Infiniband 或其他高速互联 fabric。

| 项目 | 当前情况 |
|---|---|
| 节点间互联 | 1 GbE 以太网 |
| 高速互联 | 无 Infiniband |
| 共享存储 | NFS |
| 适合任务 | 单节点或低通信量 CPU 任务 |
| 需要谨慎 | 高频通信 MPI、大规模并行 I/O、大文件频繁跨节点读写 |

## 4. 对计算任务的影响

对单节点 CPU 计算任务影响较小，例如：

- 单节点 OpenMP 程序；
- 单节点 Python / C++ / Fortran 计算；
- 单节点量子化学计算；
- 中小规模数据分析。

对以下任务需要谨慎：

- 强通信多节点 MPI；
- 多节点同时读写同一批大文件；
- 大量小文件频繁写入 NFS；
- 高频 checkpoint 写入共享目录。

如果程序支持，建议将临时文件写入本节点本地路径或任务专属目录，计算结束后再整理输出结果。

## 5. 目录组织建议

建议每个用户按项目建立目录：

```text
/data1/$USER/
├── project_a/
│   ├── input/
│   ├── scripts/
│   ├── logs/
│   └── results/
└── project_b/
    ├── input/
    ├── scripts/
    ├── logs/
    └── results/
```

这样方便定位日志、清理临时文件和向管理员反馈问题。
