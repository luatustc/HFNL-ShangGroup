## 1. 概述

ShangGroup 当前服务器为两节点 CPU SLURM 集群，包括 `node01` 和 `node02`。两台节点 CPU 与内存硬件规格基本一致，均为双路 AMD EPYC 9755 平台，每节点 256 个物理核心，适合运行 CPU 并行科学计算任务。

## 2. 节点硬件配置

| 项目 | `node01` | `node02` |
|---|---|---|
| 节点角色 | 登录节点 / Slurm 控制节点 / 计算节点 | 计算节点 |
| CPU 型号 | AMD EPYC 9755 128-Core Processor | AMD EPYC 9755 128-Core Processor |
| CPU 拓扑 | 2 路 × 128 核 × 1 线程 | 2 路 × 128 核 × 1 线程 |
| 物理核心数 | 256 | 256 |
| NUMA 节点 | 2 个 | 2 个 |
| 内存 | 约 377 GiB | 约 377 GiB |
| L3 缓存 | 约 1 GiB | 约 1 GiB |
| 操作系统 | Ubuntu 24.04.3 LTS | Ubuntu 24.04.3 LTS |
| 内核版本 | 6.17.x | 6.14.x |
| Slurm 状态 | IDLE | IDLE |

> 两节点 CPU 和内存规格一致，但内核版本不完全一致。一般用户正常提交任务即可；管理员维护时可考虑统一内核版本，减少驱动或 ABI 差异。

## 3. CPU 总资源

| 项目 | 数值 |
|---|---:|
| 节点数量 | 2 |
| 总物理核心数 | 512 |
| 每节点核心数 | 256 |
| 总内存 | 约 754 GB |
| 每节点内存 | 约 377 GiB |
| GPU | 无 |

## 4. NUMA 与并行任务建议

每台节点有 2 个 NUMA 节点：

| NUMA 节点 | CPU 核范围 |
|---|---|
| NUMA0 | 0–127 |
| NUMA1 | 128–255 |

一般用户可以先不手动绑定 NUMA，直接通过 Slurm 申请核数运行。对于对内存带宽敏感的程序，可在确认程序特性后再考虑使用 `numactl` 或进程绑定策略。

## 5. Slurm 分区设置

当前 Slurm 分区为 `debug`，包含两台节点：

| 项目 | 值 |
|---|---|
| 分区名 | `debug` |
| 是否默认 | 是 |
| 节点 | `node01`, `node02` |
| TotalCPUs | 512 |
| TotalNodes | 2 |
| 状态 | UP |
| 时间限制 | 当前为 UNLIMITED |
| OverSubscribe | NO |

可使用下面命令查看实际分区：

```bash
sinfo
scontrol show partition debug
```

## 6. CPU 作业脚本示例

示例脚本 `cpu_test.sh`：

```bash
#!/bin/bash
#SBATCH -J test_cpu
#SBATCH -p debug
#SBATCH -N 1
#SBATCH -n 64
#SBATCH -o %j.log
#SBATCH -e %j.err

set -e

echo "Job ID: $SLURM_JOB_ID"
echo "Partition: $SLURM_JOB_PARTITION"
echo "Node list: $SLURM_JOB_NODELIST"
echo "Submit dir: $SLURM_SUBMIT_DIR"
echo "Start time: $(date)"

cd "$SLURM_SUBMIT_DIR"

srun hostname

echo "End time: $(date)"
```

提交任务：

```bash
sbatch cpu_test.sh
```

查看任务：

```bash
squeue -u $USER
```

取消任务：

```bash
scancel JOBID
```

## 7. 多节点测试脚本

如果需要测试两个节点是否都能被 Slurm 调用，可以使用：

```bash
#!/bin/bash
#SBATCH -J test_2nodes
#SBATCH -p debug
#SBATCH -N 2
#SBATCH -n 8
#SBATCH -o %j.log
#SBATCH -e %j.err

set -e

echo "Job ID: $SLURM_JOB_ID"
echo "Node list: $SLURM_JOB_NODELIST"
srun hostname
```

提交后，如果日志中同时出现 `node01` 和 `node02`，说明两节点调度基本可用。
