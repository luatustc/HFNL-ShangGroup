## 1. 账号申请

如需使用实验室服务器，请联系服务器管理员（路爱同）申请账号。申请时建议说明：


- 主要计算任务类型；
- 预计需要的 CPU 核数、内存和存储空间；
- 是否需要安装额外软件。

当前服务器为纯 CPU 集群，暂无 GPU 资源。如果任务依赖 GPU，请在申请前先与管理员确认是否有其他 GPU 服务器可用。

## 2. 登录节点

当前主要登录节点为：

```text
node01
```

`node01` 同时承担登录节点、Slurm 控制节点和计算节点角色。用户登录后可以编辑代码、整理数据、提交 Slurm 作业，但不建议在登录会话中直接运行长时间或高负载计算。

## 3. SSH 登录

登录格式：

```bash
ssh 用户名@服务器地址
```

服务器地址、跳板机、VPN 或内网访问方式请通过实验室内部渠道获取。不要把真实 IP、跳板机地址、密码等敏感信息写入公开网页。

## 4. Windows 推荐工具

Windows 用户可以使用：

| 工具 | 用途 |
|---|---|
| Xshell | SSH 登录 |
| Xftp | 文件传输 |
| MobaXterm | SSH 与文件传输一体化 |
| Windows Terminal | 命令行 SSH |
| VS Code Remote SSH | 远程编辑代码 |

## 5. 文件传输

从本地上传文件到服务器：

```bash
scp 本地文件 用户名@服务器地址:/目标路径
```

从服务器下载文件到本地：

```bash
scp 用户名@服务器地址:/远程文件 本地目录
```

如果使用 Xftp 或 MobaXterm，主机地址、端口和用户名应与 SSH 登录信息保持一致。

## 6. 登录后建议先做的检查

```bash
hostname
whoami
pwd
sinfo
squeue -u $USER
df -h
```

这些命令可以帮助你确认自己登录到了哪台节点、当前有哪些 Slurm 分区、自己的任务状态以及磁盘空间情况。

## 7. 登录后不要做什么

- 不要在登录节点直接运行大规模计算；
- 不要随意删除公共目录或他人目录；
- 不要把大型临时文件长期堆在系统盘；
- 不要共享自己的账号和密码；
- 不要在公开网页、微信群、论文附录中暴露服务器真实 IP、密码或跳板机信息。
