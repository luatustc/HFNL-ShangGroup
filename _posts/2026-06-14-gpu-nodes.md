## 1. 当前结论

当前 ShangGroup 这套服务器是**纯 CPU SLURM 集群**。根据 2026-06-16 的服务器探查结果，两台节点均未检测到 `nvidia-smi`、NVIDIA GPU 设备或 GPU Slurm 资源。

因此，当前集群不适合直接运行以下任务：

- CUDA 程序；
- PyTorch / TensorFlow GPU 训练；
- 依赖 `nvidia-smi` 的 GPU 监控脚本；
- Slurm `--gres=gpu:...` 类型作业。

## 3. 用户应该怎样提交任务？

当前请按 CPU 作业提交，不要在脚本中写：

```bash
#SBATCH --gres=gpu:1
```

也不要指定不存在的 GPU 分区，例如：

```bash
#SBATCH -p gpu
```

推荐使用当前 `debug` 分区提交 CPU 任务：

```bash
#SBATCH -p debug
#SBATCH -N 1
#SBATCH -n 64
```

## 4. 如何确认是否有 GPU？

可在节点上尝试：

```bash
nvidia-smi
```

如果返回 `command not found` 或无法检测到设备，通常说明当前节点没有 NVIDIA GPU 或未安装相应驱动。

也可以查看 Slurm 资源：

```bash
sinfo -o "%P %D %c %m %G"
scontrol show node
```

如果 `%G` 显示 `(null)` 或无 GPU 资源，说明 Slurm 当前没有登记 GPU。

## 5. 如果以后加装 GPU

若后续加装 GPU，建议管理员补充以下信息：

| 项目 | 示例 |
|---|---|
| GPU 型号 | A800 / H100 / H200 / RTX 4090D 等 |
| 每节点 GPU 数量 | 例如 4 或 8 |
| 单卡显存 | 例如 80 GB |
| 驱动版本 | `nvidia-smi` 显示为准 |
| CUDA 版本 | `nvcc --version` 或模块系统显示为准 |
| Slurm GRES | `gpu:型号:数量` |
| GPU 分区 | 例如 `gpu`、`gpu_debug` |
